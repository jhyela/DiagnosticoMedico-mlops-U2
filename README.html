Objetivo de Este Repositorio Git

Repositorio para entregar las actividades de la materia Machine Learning Operations(MLOps) de la Maestria en IA - Universidad Icesi.

Descripcin del Problema

Dados los avances tecnolgicos, en el campo de la medicina la cantidad de informacin que existe de los pacientes es muy abundante. Sin embargo, para algunas enfermedades no tan comunes, llamadas hurfanas, los datos que existen escasean. Se requiere construir un modelo que sea capaz de predecir, dados los datos de sntomas de un paciente, si es posible o no que este sufra de alguna enfermedad. Esto se requiere tanto para enfermedades comunes (muchos datos) como para enfermedades hurfanas (pocos datos).

Diseo de Pipeline de ML

El siguiente modelo de pipeline de ML pretende dar solucin a los requerimientos planteados en la descripcin del problema anterior; el pipeline propuesto consiste en desplegar en produccin un modelo de Machine Learning, que, segn los datos de entrada, que en este caso sern datos relacionados con la salud y los sntomas de cada paciente, el modelo sea capaz de predecir el diagnostico de alguna de las siguientes categoras de enfermedades:

  • NO ENFERMO
  • ENFERMEDAD LEVE
  • ENFERMEDAD AGUDA
  • ENFERMEDAD CRNICA
  • ENFERMEDAD TERMINAL
Pipeline Propuesto

Fuente de Datos

Para solventar esta problemtica, tenemos como input de datos las historias clnicas de cada paciente, adicionalmente esperamos contar con el resultado de las pruebas de laboratorio, los examenes previamente realizados a igual que cualquier diagnstico mdico que se le haya realizado al paciente y que no est en su historia clnica. Por otro lado, esperamos tener acceso a bases de datos especializadas donde se encuentren diagnsticos de enfermedades raras al igual que enfermedades comunes que nos ayuden a poder clasificar los pacientes segn sus sntomas. Como base de datos de enfermedades raras se deberia tener acceso a base de datos como Orphanet o OMIM.

Almacenamiento de Datos

Debido los diversos formatos que se podran tener como datos estructurados y no estructurados, lo ms recomendable es almacenar esta informacin en un data lake.

Preparacin de los Datos

En la etapa de preparacin de datos esperamos hacer ingeniera de caractersticas (featuring engineering) y seleccin de caractersticas (feature selection) usando algoritmos tradicionales para esta tarea como PCA y LDA (anlisis de discriminante lineal).

Seleccin y Evaluacin de Modelos de ML

Para encontrar la mejor solucin para el problema planteado, se propone utilizar algoritmos como Regresin logstica mltiple, Random Forest, XGBoost, redes neuronales (MLP y Transformers) y ensambles como mquinas de soporte vectorial y GBM.

Entrenamiento, Evaluacin y Pruebas del Modelo

En esta parte de pipeline dividimos de forma aleatoria los datos de entrada en 3 partes para realizar entrenamiento, evaluacin y pruebas, generalmente la divisin de datos se realiza en porcentaje de 70%, 20% y 10% respectivamente. Para medir el desempeo de los modelos seleccionados se realizar una comparacin en mtricas como el accuracy, recall, F1-score y el MSE (Error Cuadrtico Medio).

Evaluacin de Requerimientos del Negocio

En esta parte del pipeline evaluaremos si el modelo de Machine Learning seleccionado satisface las necesidades del negocio, si la clasificacin de la gravedad de las enfermedades satisface los requerimientos de los interesados en el modelo de clasificacin; si se concluye que el modelo no satisface las necesidades del negocio nos devolveremos hasta la fase de preparacin de los datos y evalucin de caractersticas y volveremos a iniciar el proceso. Por otro lado, de ser satisfactoria la solucin brindada por el modelo de ML, seguiremos adelante con el pipeline.

Registro del Modelo de ML

Se realiza en el registro del modelo de ML para su posterior versionamiento, administracin y seguimiento. Este registro se podra realizar en herramientas como MLflow, Snowflake, Azure ML, Amazon SageMaker, GitLab o alguna otra herramienta que permita el registro del modelo de ML.

Despliegue en Produccin

Una vez el modelo de Machine Learning este entrenado, evaluado y validado, ya se encuentra listo para ser desplegado en produccin. Para esta tarea generalmente se usa uno o varios contenedores Docker y opcionalmente se puede utilizar Kubernetes en caso de tener varios contenedores Docker. Algunos de los servicios de infraetructura ms usados en la nube utilizados para desplegar modelos de ML en produccin son: Amazon web Services, Microsotf Azure y Google Cloud Platform.

Otra forma de desplegar este modelo de ML es exponiendo una API Rest para enviar datos al modelo y asi poder obtener la prediccin de la posible enfermedad del paciente.

Monitoreo

Esta tarea se realiza una vez la modelo esta desplegado en produccin y el objetivo de esta es poder monitorear si el desempeo del modelo de ML sigue cumpliendo con las expectativas de los interesados o del negocio. El desempeo del ML se puede degradar principalmente por cambios en los datos de entrada al mismo, o por cambios en los requerimientos por parte de los interesados o en algunos casos por la actualizacin de dependencias de alguno de los componentes del modelo.
